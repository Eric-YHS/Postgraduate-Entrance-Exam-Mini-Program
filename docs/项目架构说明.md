## 技术栈

| 层级 | 技术 |
|------|------|
| 后端 | Node.js 22+ / Express.js |

## 数据流

```
浏览器/小程序 → HTTP API → Express 路由 → SQLite
                              ↕
                         WebSocket ← 实时推送（通知/直播聊天）
```

### 认证流程

```
1. 用户登录 → POST /api/auth/login
2. 服务端创建 Session + 生成 Bearer Token
3. 返回 Token 给前端保存
4. 后续请求携带 Cookie 或 Authorization 头
5. /api/auth/me 复用已有 Token，避免堆积
6. Logout 时清除用户所有 Token
```

### 定时任务流程

```
cron 每分钟执行
  ├── 07:00 → dispatchDailyDigest (每日任务摘要)
  ├── 22:00 → dispatchEveningReminder (未完成任务晚间提醒)
  └── 每分钟 → dispatchDueTaskReminders (任务开始提醒)
```

## 设计决策

1. **单体 server.js**：所有路由在一个文件中，简单直接，适合当前规模
2. **SQLite 而非 MySQL/PostgreSQL**：零配置、单文件、便于备份迁移
3. **原生 JS 而非框架**：无构建步骤、部署简单、符合小程序风格
4. **Session + Token 双认证**：Web 用 Session，API/小程序用 Bearer Token
5. **邻接表文件夹**：利用 SQLite 递归 CTE 实现树形结构，简单高效
6. **SM-2 算法**：成熟的间隔重复算法，与 Anki 相同
