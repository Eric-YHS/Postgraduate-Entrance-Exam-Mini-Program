# 考研规划小程序客户采购与开通指南

更新时间：2026-03-12

适用范围：本指南用于“考研规划小程序”正式上线前，协助贵司完成服务器、域名、小程序主体、备案与 HTTPS 的前置准备。  
说明：本指南不是把工作推给贵司，而是这些环节本身就涉及企业主体实名、账号归属、备案合规和后期资产所有权，原则上应由企业主体自行申请和持有，我们可以全程提供远程指导和技术配合。

## 1. 为什么建议由贵司自行购买和申请

这一步不是因为技术方不愿意协助，而是因为以下事项天然应归企业主体本人或企业授权负责人办理：

1. 云账号实名认证会直接影响账号和云资源归属。腾讯云官方明确说明，实名认证直接影响账号和资源归属，企业用户如果使用个人信息实名认证，后续人员变动或账号纠纷时，可能影响业务并造成损失。
2. 域名实名认证关系到域名所有权。域名后续续费、过户、备案、解析、证书申请都与域名实名信息绑定，不建议由第三方个人代持。
3. ICP 备案需要提交企业主体信息，并且包含人脸核验、短信核验、主体负责人/服务负责人信息。部分省市还要求主体负责人必须为法定代表人，或者要求提供授权书、加盖公章等材料。
4. 本项目的小程序方案包含 `web-view` 页面用于打开直播/网页内容。微信开放文档明确说明，个人类型的小程序暂不支持使用 `web-view`，因此正式上线建议使用企业主体或符合条件的个体工商户主体。
5. 后续如果发生人员更替、服务商更换、域名迁移、服务器扩容、数据合规审查，企业自己持有账号会更安全，也更方便。

结论：  
贵司应自行注册并持有云账号、域名账号、小程序主体账号；技术方负责提供清单、选型建议、远程指导、部署实施和联调支持。

## 2. 我们会配合到什么程度

我们可以提供以下配合：

1. 提供购买建议和配置清单。
2. 远程指导贵司完成腾讯云、小程序账号、域名购买与备案流程。
3. 在贵司完成购买后，负责项目部署、Nginx 配置、HTTPS 部署、接口联调和上线前检查。
4. 协助贵司完成微信小程序后台的服务器域名、业务域名等技术配置。

我们不建议做的事情：

1. 由技术方个人账号代替企业购买服务器或域名。
2. 由技术方代持小程序主体账号。
3. 在企业主体尚未明确的情况下先用个人主体上线正式版本。

## 3. 贵司需要提前准备的资料

建议先准备以下资料，再开始操作：

1. 企业营业执照扫描件或清晰照片。
2. 法定代表人身份证件。
3. 企业常用手机号、企业邮箱。
4. 可接收备案短信和进行人脸核验的负责人手机。
5. 公司全称、统一社会信用代码、注册地址。
6. 如非法定代表人代办，准备授权书和加盖公章材料。

说明：不同省通信管理局要求可能不同，备案时以贵司所在地管局要求为准。

## 4. 推荐的办理顺序

建议按下面顺序进行，不要跳步：

1. 注册腾讯云企业账号并完成企业实名认证。
2. 购买云服务器。
3. 购买域名并完成域名实名认证。
4. 注册微信小程序账号，拿到 `AppID`。
5. 完成 ICP 备案。
6. 申请并部署 SSL 证书。
7. 把账号信息和资源信息交给技术方部署。

## 5. 具体操作步骤

## 5.1 注册腾讯云企业账号

建议使用贵司自己的手机号和邮箱注册腾讯云主账号，并完成企业实名认证。

建议负责人：

1. 法定代表人本人。
2. 法定代表人明确授权的行政/IT/运营负责人。

不建议：

1. 使用外包人员私人手机号注册主账号。
2. 使用离职风险较高的个人邮箱作为唯一主账号。

操作建议：

1. 打开腾讯云官网：<https://cloud.tencent.com/>
2. 点击注册，创建贵司自己的腾讯云账号。
3. 完成企业实名认证，不要使用个人实名认证。
4. 认证通过后，再进行服务器和域名购买。

## 5.2 购买服务器

本项目当前建议购买 Linux 服务器，后续由我们负责部署。

推荐配置：

| 项目 | 建议 |
| --- | --- |
| 云厂商 | 腾讯云 |
| 服务器类型 | 云服务器 CVM |
| 操作系统 | Ubuntu 22.04 LTS |
| CPU / 内存 | 2 核 4 GB 起步 |
| 系统盘 | 50 GB SSD 起步 |
| 公网带宽 | 5 Mbps 起步 |
| 地域 | 选择中国大陆地域，优先离主要用户近的城市 |
| 购买时长 | 先买 1 年 |

为什么这样选：

1. 当前项目包含学生端小程序、老师端网页后台、文件上传、定时任务、论坛、商城等模块，`2 核 4 GB` 是更稳妥的起步配置。
2. 使用中国大陆地域更适合后续备案和国内访问速度。
3. Ubuntu 22.04 LTS 兼容性更好，便于后续 Docker 和 Node.js 部署。

购买时请注意：

1. 要有公网 IP。
2. 保留服务器登录方式信息。
3. 不需要先安装业务环境，我们后续统一部署。

## 5.3 购买域名

建议购买一个与品牌相关的 `.com` 或 `.cn` 域名，并使用企业信息实名。

建议规则：

1. 域名尽量简短、好记。
2. 尽量与考研规划品牌名一致。
3. 建议由企业作为域名实名主体，不建议挂在个人名下。

购买步骤：

1. 登录腾讯云域名注册页面。
2. 先创建企业信息模板。
3. 使用企业信息模板购买域名。
4. 完成域名实名认证。

注意：

1. 腾讯云官方说明，注册域名前需要先完成账号实名认证。
2. 域名注册和后续备案时，实名信息应尽量与企业主体保持一致。
3. 部分省管局对“域名所有者”和“备案主体”的一致性有明确要求，具体以当地规则为准。

## 5.4 注册微信小程序账号

正式上线前，需要由贵司自己注册小程序主体账号。

建议主体：

1. 企业主体优先。
2. 如贵司是个体工商户，也建议按合规主体办理。

不建议：

1. 用个人类型小程序作为正式长期方案。

原因：

1. 微信开放文档说明，`web-view` 组件“个人类型的小程序暂不支持使用”。
2. 本项目当前方案中，小程序会用到网页承载能力，因此企业主体更稳妥。

注册后请提供：

1. 小程序 `AppID`
2. 管理员信息
3. 是否已完成认证

## 5.5 ICP 备案

如果服务器放在中国大陆并准备正式给国内用户访问，ICP备案基本是必须环节。

为什么一定要企业自己参与：

1. 备案需要填写备案主体信息。
2. 备案流程包含人脸核验。
3. 备案流程包含短信核验。
4. 备案中会涉及主体负责人、服务负责人、法定代表人等信息。
5. 部分地区要求主体负责人必须为法定代表人，或需上传授权书、加盖公章材料。

建议做法：

1. 使用与服务器同主体的腾讯云账号发起备案。
2. 域名实名信息尽量与备案主体保持一致。
3. 按腾讯云备案流程一步步提交，不要找第三方个人代备案。

备案前要知道：

1. 备案审核需要时间，不能当天完成。
2. 不同省份要求不同。
3. 备案完成后才能更稳妥地配置正式域名访问。

## 5.6 申请 HTTPS 证书

小程序正式调用接口时，必须使用 HTTPS；如果后续继续使用网页承载页面，也同样建议全站 HTTPS。

建议：

1. 先申请域名型 DV 证书即可满足起步上线。
2. 后续若贵司有更高品牌展示或合规要求，再考虑 OV/EV。

说明：

1. 微信开放文档明确要求小程序请求域名使用 HTTPS/WSS。
2. 小程序后台配置服务器域名时，不能使用 IP 地址或 `localhost`。
3. 证书签发和部署可以由我们配合完成，但证书申请主体和域名管理权限仍应掌握在贵司手里。

## 6. 购买完成后，请把这些信息交给我们

贵司完成购买和申请后，请统一提供以下信息：

1. 服务器公网 IP。
2. 服务器系统版本。
3. 服务器登录方式。
4. 域名名称。
5. 域名 DNS 管理权限，或者由贵司配合添加解析记录。
6. 小程序 `AppID`。
7. 小程序后台管理员是否可以配合配置服务器域名、业务域名。
8. ICP 备案号（备案完成后）。
9. SSL 证书申请状态。

为了安全，建议优先采用以下协作方式之一：

1. 由贵司创建专用运维账号给我们。
2. 由贵司临时共享远程操作窗口，由我们指导贵司操作。
3. 在贵司内部人员在场的情况下完成敏感配置。

## 7. 给贵司的直接说明话术

下面这段可以直接发给客户：

> 我们这边会全程提供选型建议、购买清单、远程指导、部署实施和联调支持，但账号本身最好由贵司企业主体本人或授权负责人来开通，这样后续不管是续费、换供应商、做备案变更还是内部交接，都会更安全、更规范。

## 8. 常见问题

### 8.1 能不能先让技术方个人代买，后面再转给企业？

不建议。  
腾讯云官方说明，实名认证会影响账号和资源归属；域名所有者也直接关系到域名拥有权。后期即使可以变更，也会增加流程成本、合规风险和交接风险。

### 8.2 能不能先不用备案？

如果服务器放在中国大陆并准备正式服务国内用户，通常不能把“长期不备案”作为正式方案。  
建议尽早备案，避免后续域名接入、正式访问和小程序联调受影响。

### 8.3 小程序能不能先用个人主体？

不建议作为正式方案。  
当前项目方案包含 `web-view` 能力，微信开放文档说明个人类型小程序暂不支持使用该组件。

### 8.4 你们这边还能怎么配合？

可以。  
贵司负责“开户注册和实名持有”，我们负责“技术实施和上线支持”，这是最稳妥的协作方式。
