## 4. 腾讯云控制台 VNC 登录

腾讯云页面里的“VNC 登录”是浏览器内置控制台，不要求服务器额外安装独立 VNC 服务。即使系统没有桌面环境，也可以通过该入口进入系统控制台。

操作步骤：

1. 打开腾讯云实例详情页
2. 点击“登录”
3. 选择“VNC 登录”
4. 在登录界面输入用户名 `ubuntu`
5. 输入系统密码 `Shi0418!`

补充说明：

- 图中提示“初始登录名为 ubuntu”，当前实例与该提示一致
- 如果 SSH 能登录，但网页 VNC 提示密码错误，可先在腾讯云控制台执行“重置密码”
- 重置的是系统账号密码，通常会同时影响 SSH 和控制台登录

## 5. 常见问题

### 5.1 SSH 连接超时

优先检查：

- 实例是否处于“运行中”
- 腾讯云防火墙 / 安全组是否放行 `TCP 22`
- 本地网络是否拦截了外连 `22` 端口

### 5.2 Password denied / 权限错误

确认以下信息是否一致：

- 用户名：`ubuntu`
- 密码：`Shi0418!`

如果仍然失败，可在腾讯云控制台重置密码后再试。

### 5.3 VNC 能进，SSH 进不去

通常说明实例存活，但 SSH 服务、22 端口或防火墙配置有问题。可先通过 VNC 进入系统后检查：

```bash
sudo systemctl status ssh
sudo ss -lntp | grep :22
```

### 5.4 需要图形化远程桌面

当前已验证的是：

- SSH 远程终端登录
- 腾讯云控制台的 VNC/控制台登录

如果要使用真正的图形桌面远程访问，还需要额外安装桌面环境以及 `xrdp` 或独立 VNC 服务，当前文档不包含该部分部署。
